Lighthouse User Guide

Track port usage, find conflicts, and understand who owns what.

What Lighthouse does

  • Scans live listening ports on your machine
  • Finds PORTMASTER.md files across agent rules and repositories
  • Helps you spot port conflicts quickly
  • Suggests nearby free ports when a port is already in use

How to use the main table

  • Click Scan to refresh the live machine state
  • Use the filter buttons to narrow results by source
  • Click any column header to sort ascending or descending
  • Service and Managed By are especially useful for alphabetical review

What the columns mean

  • Port: the listening port number
  • Bind: where the service is listening, such as 127.0.0.1 or 0.0.0.0
  • Service: known service or project name when available
  • Process: live process name currently listening
  • Source: whether the row came from a live scan, Docker, PORTMASTER, or project config
  • Managed By: the owning runtime, app, or tool when known

Quick Check

  • Enter a port number to see if it is currently in use
  • Use Suggest Free to find the next available port in the app-server range

Conflict workflow

  1. Scan the machine
  2. Open the Conflicts section if warnings appear
  3. Review the affected processes and suggested replacement port
  4. Open the related config or PORTMASTER entry and update intentionally

Keyboard shortcuts

  • Escape: close the guide modal
  • Enter on Guide or Scan buttons: activate them
